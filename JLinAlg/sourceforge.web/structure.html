JLinAlg - structure: packages and classes

The API

All packages of JLinAlg have the prefix org.jlinalg . The API's main classes are under this package:
    • The definition of ring elements in class RingElement,
    • the corresponding factory RingElementFactory, and
    • the interfaces (with the prefix I) for both.
  • The classes defining matrices and vectors.
  • Utilities such as a linear solver, optimized matrix multiplication, and other supporting classes.
All new data types must implement the interface IRingElement (typically, they extend on FieldElement, which extends on RingElement and implements IRingElement ) and provide a factory implementing IRingElementFactory (by usually extending on RingElementFactory ). Realizations of ring elements are defined in the packages
  • org.jlinalg.polynomial permits the creation of generic polynomials.
  • org.jlinalg.rational implements rational numbers based on java.util.BigIntegers.
  • org.jlinalg.complex implements complex numbers based on Rationals.
  • org.jlinalg.f2 is the Galois field with two values.
  • org.jlinalg.doublewrapper is a wrapper for java.lang.Double.
  • org.jlinalg.bigdecimalwrapper similar, but for java.util.BigDecimal.
  • org.jlinalg.field_p is a generic definition of Galois fields.
The packages
  • org.jlinalg.operator define operators.
  • org.jlinalg.latex allows to create chunks of LaTeX from variables, matrices, and so on.
  • org.jlinalg.optimise implements a simple optimizer (slow but general).

The tests

Tests are in the same packages as their API-counterparts. The tests are implemented under the JUnit4 framework and, by convention, their class namesfollow the pattern XxxTest . They are grouped into test suites (by convention their class names are XxxTests) ; most are located in package org.jlinalg.tests . Among them are:
  1. UnitTests which comprises the usual unit tests,
  2. BlackBoxTests which includes more expensive tests (e.g. tests using random numbers, large matrices, ...)
  3. AllTests which combines the above as well as tests for the demos and tests for the tests.
Code supporting the tests is located in the package org.jlinalg.testutil. The classes XxxTestBase correspond to classes in org.jlinalg . These classes are meant to be extended by classes representing a new data type. Methods in these classes (by convention having the postfix _base ) should usually not be overwritten as to prevent the the basic tests are canceled. In certain cases, these tests may fail even though the implementation of the base type is correct (and then overwriting is meaningful). When a new test class extends one of the tests, it must implement the interface org.jlinalg.testutil.TestBaseInterface . As several of the tests create instances of ring elements from strings representing integer numbers, the method XxxFactory.get(Object o) must accept strings.

The classes StringWrapperXxx are used in some tests.

The demos

The demos are in package org.jlinalg.demo . At the moment, a few basic tests use the "depreciated" annotation to decide whether to run a specific test or not. Experimentally, annotation org.jlinalg.JLinAlgTypeProperties is used to determine whether certain test are executed and/or their configuration and/or their expected outcomes.

The test suite is modified from JUnit: see org.jlinalg.testutils.SuiteJLinAlg
